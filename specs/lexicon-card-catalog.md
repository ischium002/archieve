## 已拍板的决定

1. **原 world card 去留**：Universe 页保留前三张综述卡（时代背景 / 魔法体系 / 科技背景），恶魔族群、HEAD-LESS、天使族群三张词条卡迁入词条馆，原位换一张入口卡。
2. **翻卡形态**：堆叠露顶边（每张卡只露顶端一行：词条名 + 编号），不用网格平铺。
3. **入口**：nav 加 Lexicon 项 + Universe 页入口卡，两个都要。
4. **文案分工**：迁移的三张卡沿用坐骨原文；新词条列表和文案由坐骨提供，壳先用 `draft: true` 占位词条撑结构。
5. **彩蛋**：页面初载抽屉是关着的（只见抽屉面板 + 铜牌标签），点一下拉开。深链直达词条时跳过关抽屉状态。

## 页面结构（`lexicon/index.html` + `lexicon/lexicon-data.js`）——2026-07-19 版

- **柜面**：一只多屉目录柜。柜顶铜牌行（LEXICON · CARD CATALOG / 查卡 ⌕ / 目录柜 No.03），下面**每个分类一只抽屉**：分类名 + mono 计数（N 张 · M 待归档）+ 铜色杯式拉手。点抽屉面板拉开/推回（grid-template-rows 0fr↔1fr 动画）。
- **屉内**：立式索引卡，竖排互叠只露顶边（负 margin），卡头一条制式红线；draft 占位卡虚线边框 + 「待归档」小签，不可点。屉底一根铜色压卡横杆。hover 微抬，卡堆内 ↑↓ 键移动焦点。
- **查卡（检索）**：点铜牌上「查卡 ⌕」滑出隐藏检索屉，按词条名/sub/def 过滤：命中卡正常、未命中卡变淡，有命中的抽屉自动拉开、全空的抽屉面板变淡。清空恢复检索前的开合状态。
- **抽出态**：点某张卡，放大到屏幕中央 overlay。仿目录卡：词条名下一条红头线、正文横线格（行高与格线同步 28px）、右上角 mono 编号、分类小印章（旋转、`--stamp` 红）、编者注（灰斜体 ✎ 前缀）、参见互链（点击换卡：先插回再抽出的 swap 动画）、**同分类相邻词条 ←→ 翻阅**（按钮 + 键盘方向键）、底部中央穿杆圆孔。关闭：× / 点背景 / Esc。
- 初版「整柜闭合态彩蛋」由多屉形态天然继承（初载所有抽屉都关着）；console 馆员彩蛋保留。

## 数据 schema（`lexicon-data.js`）

```js
LEX_CATS    // 有序数组 [{ key, label }]，顺序 = 分隔签顺序
LEX_ENTRIES // [{ id, term, sub?, cat, def, note?, see?, draft? }]
```

- `id` 是锚点路由：`lexicon/#<id>`，起了就别改。全站术语从别处链过来就用这个。
- 编号 `LEX-001` 按数组下标自动生成，**含 draft 占位**——填完文案把 `draft` 去掉即上架，编号不动。缺号 = 待归档，页脚有一句解释（这是特性不是 bug）。
- `see` 指向 draft 词条时渲染为纯文本 +「（待归档）」，不成死链。
- 没有非 draft 词条的分类不渲染分隔签。

## 路由

- `#<id>` 直达：打开即抽屉展开（无动画）+ 该卡抽出。
- 抽卡写 hash，关卡清 hash（replaceState），hashchange 统一驱动，前进后退可用。
